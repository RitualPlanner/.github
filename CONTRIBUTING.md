# Contributing to RitualPlanner

Thank you for contributing to RitualPlanner.

## Before You Start

- Search existing Issues.
- Discuss major changes before implementation.

## Development Workflow

1. Fork or create a new branch.
2. Use descriptive branch names.
3. Follow Conventional Commits.
4. Submit a Pull Request.

## Branch Naming

```
feat/login

feat/dashboard

fix/auth

fix/payment

docs/readme
```

## Commit Format

```
feat: add JWT refresh token

fix: resolve login issue

docs: update installation guide
```

## Code Style

- Write clean, readable code.
- Keep functions small.
- Use TypeScript strict mode.
- Follow ESLint rules.

## Pull Requests

Every Pull Request must

- pass CI
- include testing
- include documentation when necessary

Thank you for helping improve RitualPlanner.
